# Documento 013 — Design System

**Categoria:** Produto  
**Versão:** 1.0  
**Status:** Aprovado  
**Última atualização:** 13/07/2026  
**Responsável:** Elberth Moraes

---

> Este documento define os princípios de experiência do usuário, linguagem e identidade visual do **Comunexa Escola**.

Seu objetivo é garantir que todas as telas, mensagens e interações transmitam uma experiência simples, consistente e acolhedora.

Este documento representa a primeira versão do Design System da Comunexa.

---

# Objetivo

Garantir que o sistema seja intuitivo, consistente e fácil de utilizar por qualquer pessoa, independentemente do seu conhecimento técnico.

A experiência do usuário deverá ser considerada tão importante quanto as funcionalidades implementadas.

---

# Nossa Comunicação

O sistema deverá transmitir:

- organização;
- confiança;
- simplicidade;
- acolhimento.

As mensagens deverão ser:

- curtas;
- diretas;
- positivas;
- humanas;
- fáceis de compreender.

Sempre que possível utilizaremos uma linguagem próxima daquela utilizada pelas escolas.

---

# Público

O sistema será utilizado por:

- Direção;
- Coordenação;
- Secretaria;
- Professores;
- Responsáveis;
- Alunos (futuramente).

Toda comunicação deverá ser acessível para esses públicos.

---

# Vocabulário Oficial

| Utilizar | Evitar |
|-----------|---------|
| Publicação | Postagem |
| Comunicado | Mensagem |
| Tarefa | Dever |
| Prova | Avaliação *(quando se tratar de prova comum)* |
| Evento | Programação |
| Responsável | Cliente |
| Turma | Classe |
| Escola | Instituição *(na interface)* |
| Publicar | Enviar |
| Atualizar | Alterar |
| Arquivar | Excluir *(sempre que possível)* |

---

# Botões

Utilizar verbos simples.

Exemplos:

- Publicar
- Salvar
- Atualizar
- Arquivar
- Compartilhar
- Copiar Link
- Baixar Arquivo

Evitar:

- Executar
- Processar
- Confirmar Operação
- Efetivar

---

# Mensagens de Sucesso

As mensagens deverão ser objetivas.

Exemplos:

- Publicação criada com sucesso.
- Alterações salvas.
- Mensagem copiada.
- Arquivo enviado.

---

# Mensagens de Erro

Nunca culpar o usuário.

Exemplos:

- Não foi possível salvar a publicação.
- Tente novamente em alguns instantes.
- Alguns campos precisam ser preenchidos.

Evitar mensagens técnicas.

Sempre que possível indicar como resolver o problema.

---

# Campos Obrigatórios

Os campos obrigatórios deverão ser identificados claramente.

Exemplo:

```text
Título *
```

Evitar mensagens genéricas como:

> Campo inválido.

Preferir:

> Informe o título da publicação.

---

# Datas

Utilizar formatos amigáveis.

Exemplos:

- Hoje
- Ontem
- Amanhã
- 15 de março
- Sexta-feira, 20 de abril

Evitar datas exclusivamente numéricas quando houver espaço suficiente.

---

# Estrutura das Publicações

Toda publicação deverá possuir:

- título claro;
- resumo curto;
- conteúdo organizado;
- autor;
- data de publicação;
- data da última atualização.

---

# Mensagens para WhatsApp

As mensagens deverão ser curtas e objetivas.

Exemplo:

```text
📢 Novo comunicado

Foi publicado um novo comunicado para a Turma 901.

Confira todas as informações:

🔗 https://...
```

Evitar mensagens longas.

O objetivo do WhatsApp é direcionar o usuário para a publicação oficial.

---

# Organização Visual

A interface deverá transmitir leveza.

Princípios:

- poucos botões;
- poucos menus;
- poucas cores;
- bastante espaço em branco;
- leitura rápida;
- boa hierarquia visual.

---

# Experiência do Usuário

Antes de disponibilizar qualquer funcionalidade, responder:

> O usuário consegue realizar essa tarefa sem treinamento?

Se a resposta for negativa, a funcionalidade deverá ser simplificada.

---

# Personalidade da Comunexa

Queremos que o produto seja percebido como:

- simples;
- confiável;
- organizado;
- moderno;
- acolhedor;
- intuitivo.

Não queremos que seja percebido como:

- burocrático;
- complexo;
- pesado;
- técnico;
- confuso.

---

# Princípios do Design

Toda tela deverá seguir os seguintes princípios:

- clareza antes da criatividade;
- consistência antes da personalização;
- simplicidade antes da complexidade;
- acessibilidade para todos os públicos;
- foco na tarefa principal.

---

# Evolução do Design System

Este documento representa a primeira versão do Design System da Comunexa.

No futuro poderão ser incorporados:

- identidade visual;
- paleta de cores;
- tipografia;
- componentes;
- biblioteca de ícones;
- padrões de formulários;
- padrões de navegação;
- acessibilidade;
- modo escuro;
- design responsivo.
